P01 Dashboard / NEW

Landing page after login. Single-glance overview of everything happening across the tenant's bots, leads, and conversations.

KPI Cards Row (top)
Active Bots
Count of bots with status WORKING. Click drills to Bots page.
Open Conversations
Conversations with status=open. Click drills to Conversations page.
New Leads (7d)
Leads created in last 7 days. Click drills to Leads page.
Conversion Rate
(qualified + converted) / total leads as percentage.
Messages Today
Inbound + outbound messages in current day.
Active Workflows
Workflow executions with status=active.
Main Content
Activity Feed
Chronological list of recent events (new lead, message received, workflow completed, bot connected). Each row links to the relevant detail page.
Lead Pipeline Funnel
Horizontal bar chart: new → contacted → qualified → converted. Shows count at each stage.
Bot Health Grid
Mini cards for each bot showing connection status (green/yellow/red dot), last message time, and message count today.
Messages Over Time
Line chart (Recharts) showing inbound vs outbound messages over last 7/30 days.

P02 Bots / Workspaces /bots EXISTS

Manage WhatsApp bot instances. Create, configure, monitor connection status, and delete bots. Already built as BotsPage.tsx.

Current Features (already built)
  • Grid of BotCards showing name, status, phone, AI engine, active leads, volume
  • "Add New Bot" button — creates Evolution API instance, shows QR code modal
  • BotSetupModal — polls for QR code, shows connection status, edit name/system prompt
  • Delete bot with confirmation
  • Backend health indicator
Still Needed
  • Bot detail view — click a card to see full config, conversation list, lead list for that bot
  • Edit bot config inline (system prompt, AI model, temperature)
  • Bot-level analytics — messages/day, response time, lead conversion
  • Start/stop/restart controls
  • Bulk actions (select multiple → delete/stop)
P03 Conversations /conversations CRITICAL NEW

WhatsApp-style inbox. See all active conversations, read message history, and send replies. This is the core operational page for agents.

Layout — 3-Panel
Left Panel — Conversation List
  • Search bar (by name, phone, or message content)
  • Filter tabs: All / Open / Closed / Unread
  • Conversation rows: avatar/initials, name, last message preview, timestamp, unread badge
  • Sorted by lastMessageAt descending
  • Color-coded status dot (green=open, gray=closed)
Center Panel — Message Thread
  • Header: contact name, phone, lead status badge, linked bot name
  • Message bubbles — outbound (right, green), inbound (left, dark)
  • Timestamps between message groups
  • Message types: text, image thumbnail, audio player, location map link
  • Auto-scroll to bottom on new messages
  • "AI is typing..." indicator when bot is processing
Right Panel — Contact Sidebar
  • Lead info: name, phone, email, source, status
  • Quick status change dropdown (new → contacted → qualified → converted)
  • Attributes list (budget, timeline, company, etc.)
  • Active workflow indicator
  • Conversation metadata: platform, created date, message count
  • "Assign to agent" button
  • Notes section (agent can add internal notes)
Message Input Bar (bottom of center panel)
  • Text input with placeholder "Type a message..."
  • Attach media button (image, audio, document, location)
  • Emoji picker
  • Quick reply templates dropdown
  • Send button (Enter to send, Shift+Enter for newline)
  • Toggle: "Bot auto-reply ON/OFF" per conversation
Data Sources
  • GET /api/conversations — list with last message preview
  • GET /api/conversations/:id/messages — full message history
  • POST /api/conversations/:id/send — send agent reply
  • WebSocket — real-time incoming messages via Socket.IO

P04 Leads /leads CRITICAL NEW

CRM lead management. View all leads, filter by status/source/bot, edit details, and track through the pipeline.

View Modes (toggle)
Table View (default)
Sortable columns: Name, Phone, Source, Status, Bot, Created, Last Activity. Pagination. Bulk select for export/delete.
Kanban View
Columns by status: New → Contacted → Qualified → Converted → Lost. Drag-and-drop to change status. Cards show name, phone, source badge.
Filters Bar
  • Status multi-select: New, Contacted, Qualified, Converted, Lost
  • Source filter: WhatsApp, Website, Referral, Manual
  • Bot filter: which bot captured the lead
  • Date range: created between X and Y
  • Search: name, phone, email, or attribute value
Lead Detail Slide-over (click a row)
  • Contact info: name, phone, email, source
  • Status dropdown with pipeline position
  • Attributes editor — add/edit/delete custom key-value pairs
  • Linked bot and conversation (click to open Conversations page)
  • Workflow history — which workflows ran, current step, collected data
  • Timeline — every touchpoint: messages sent/received, status changes, workflow events
  • Notes — agent notes with timestamps
  • AI Score — if lead qualifier bot ran, show score and reasoning
Actions
  • Import leads (CSV upload)
  • Export leads (CSV/JSON)
  • Add lead manually
  • Delete leads (bulk)
  • Assign to agent

P05 Workflows /workflows NEW

Multi-step conversational flows. Define guided conversations that bots walk leads through (qualification, onboarding, support triage).

Workflow List Page
  • Card grid showing each workflow: name, trigger intent, step count, active executions count
  • "Create Workflow" button opens builder
  • Duplicate / Delete / Archive actions per workflow
Workflow Builder (detail page or modal)
Step Editor
  • Vertical timeline of steps
  • Each step: key (variable name), prompt (what bot says), expected input type
  • Conditions: branch logic (if budget > $50k → skip to step 4)
  • Drag to reorder steps
  • Add/remove steps
Trigger Config
  • Trigger type: keyword match, intent, manual, new conversation
  • Keywords: comma-separated trigger words
  • Bot assignment: which bot runs this workflow
  • Fallback: what happens if workflow fails/times out
Execution Monitor
  • Table of active/completed executions
  • Columns: Lead name, current step, status, started at, duration
  • Click to see collected data at each step
  • Retry / Cancel / Reset execution

P06 Contacts /contacts NEW

Address book of all WhatsApp contacts. Unlike Leads (which tracks pipeline status), Contacts is a directory of everyone the bots have interacted with.

Contact List
  • Searchable table: name, phone, tags, last interaction, linked lead status
  • Auto-populated from WhatsApp conversations
  • Manual add/edit for non-WhatsApp contacts
  • Tags: custom labels (VIP, enterprise, churned, etc.)
  • Bulk operations: tag, export, delete
Contact Detail
  • Full contact info with edit capability
  • Conversation history across all bots
  • Lead record link (if exists)
  • Custom fields (key-value pairs)
  • Notes with timestamps
P07 Analytics /analytics NEW

Data-driven insights. Charts, metrics, and reports across bots, conversations, leads, and AI usage.

Dashboard Sections
Messaging Metrics
  • Messages over time (line chart, inbound vs outbound)
  • Messages by bot (stacked bar)
  • Average response time
  • Busiest hours heatmap
Lead Metrics
  • Leads over time (line chart)
  • Pipeline funnel (new → converted)
  • Conversion rate by bot
  • Lead source breakdown (pie chart)
  • Average time to conversion
AI Performance
  • AI token usage over time
  • Cost breakdown by model
  • Workflow completion rate
  • Fallback/escalation rate
Bot Performance
  • Uptime per bot
  • Conversations per bot
  • Leads generated per bot
  • Error rate
Controls
  • Date range picker (7d, 30d, 90d, custom)
  • Bot filter (all or specific)
  • Export charts as PNG/CSV
P08 Settings /settings NEW

Tenant configuration, user management, API keys, billing, and integrations.

Tabs
General
  • Tenant name
  • Default AI model (Groq/OpenAI/Gemini)
  • Default system prompt template
  • Timezone
  • Language
Team
  • User list: email, role, status, last active
  • Invite user (email + role)
  • Change role / Remove user
  • Roles: Admin, Agent, Viewer
API Keys
  • List of API keys: name, created, last used
  • Generate new key (shows once, copy to clipboard)
  • Revoke key
Integrations
  • Evolution API connection config
  • Groq API key
  • OpenAI API key
  • Webhook URL config
  • Chatwoot integration toggle
Billing
  • Current plan (Free/Pro/Enterprise)
  • Usage meters: messages, AI tokens, storage
  • Invoices list
  • Upgrade/downgrade buttons
Danger Zone
  • Delete tenant (irreversible, type confirm)
  • Export all data (JSON/CSV)
P09 Login / Register /login, /register CRITICAL NEW

Authentication pages. Currently the backend has auth routes but the frontend has no login flow — it uses a hardcoded API key.

Login Page
  • Email + Password fields
  • "Remember me" checkbox
  • "Forgot password?" link
  • Submit → POST /api/auth/login → store JWT in httpOnly cookie or localStorage
  • Redirect to Dashboard on success
Register Page
  • Company name, Email, Password, Confirm Password
  • Creates tenant + admin user in one flow
  • Submit → POST /api/auth/register → auto-login → redirect to Dashboard
Auth Flow
User visits / → No JWT? → Redirect /login → Login OK → Store token → Redirect /dashboard
