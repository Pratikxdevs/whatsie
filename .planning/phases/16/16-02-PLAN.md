  <action>
Create `src/normalizer/discord.ts` with a single exported function `normalizeDiscordMessage(tenantId: string, message: Message): NormalizedMessage` where `Message` is imported from discord.js.

The function must:
1. Check `message.author.bot` at the top -- if true, throw `new Error('Ignoring bot message')` to prevent infinite loops (per Pitfall 2 in research).
2. Determine message `type` from attachments: iterate `message.attachments`, classify by mimeType prefix (image/* -> 'image', video/* -> 'video', audio/* -> 'audio', else -> 'file'). Default to 'text' if no attachments.
3. Build the attachments array with `{ type, url: attachment.url, mimeType: attachment.contentType || 'application/octet-stream', size: attachment.size, fileName: attachment.name }`.
4. Extract mentions from `message.mentions.users` as an array of user ID strings.
5. Return a NormalizedMessage with:
   - `platform: 'discord'`
   - `userId: message.author.id` (Discord snowflake string)
   - `message.text`: `message.content || null`
   - `message.attachments`: built array
   - `message.quickReplies`: null
   - `type`: determined type
   - `timestamp`: `new Date(message.createdTimestamp).toISOString()`
   - `metadata.raw`: object with `{ id: message.id, channelId: message.channelId, guildId: message.guildId, authorId: message.author.id, authorUsername: message.author.username, authorBot: message.author.bot }`
   - `metadata.replyTo`: `message.reference?.messageId || null`
   - `metadata.isForwarded`: false
   - `metadata.mentions`: extracted mentions array
   - `metadata.channelId`: `message.channelId`
   - `metadata.guildId`: `message.guildId || null`
   - `metadata.messageId`: `message.id`
   - `metadata.authorUsername`: `message.author.username`
  </action>
