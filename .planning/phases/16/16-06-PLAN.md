      code,
      redirect_uri: process.env.DISCORD_OAUTH_REDIRECT_URI!,

      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },

        username: process.env.DISCORD_CLIENT_ID!,

**In src/routes/workspaces.ts:**
Add Discord OAuth2 routes following the Telegram userbot flow pattern.

**POST /api/workspaces/discord/callback** -- OAuth2 callback endpoint:
1. Accept `{ code, state }` in request body (state encodes tenantId for CSRF protection).
2. Exchange code for token via POST to `https://discord.com/api/v10/oauth2/token`:
   - Content-Type: `application/x-www-form-urlencoded`
   - Basic auth: `DISCORD_CLIENT_ID:DISCORD_CLIENT_SECRET`
   - Body: `grant_type=authorization_code&code=${code}&redirect_uri=${DISCORD_OAUTH_REDIRECT_URI}`
3. The OAuth2 response with `bot` scope includes a `bot` object with `bot.token`. Extract `bot_token` from response.
4. Fetch bot user info via GET `https://discord.com/api/v10/users/@me` with `Authorization: Bot ${bot_token}`.
5. Validate bot_token by calling `DiscordAdapter.connect(bot_token, tempId)` with a 10s timeout.
6. Create DB record: `prisma.bot.create(...)` with `platform: 'discord'`, `status: 'connected'`, `sessionName`, config containing `bot_token` and `bot_token_masked`.
7. Reconnect with actual bot.id, attach messageCreate handler using `attachDiscordMessageHandler(bot.id, tenantId)`.
8. Return `{ status: 'success', workspace: mapBotToWorkspace(bot) }`.
9. On any error: return 502 with error details.

**GET /api/workspaces/discord/oauth-url** -- Generate OAuth2 URL:
1. Accept optional `state` query parameter.
2. Build URL: `https://discord.com/oauth2/authorize?client_id=${DISCORD_CLIENT_ID}&scope=bot&permissions=2147483648&redirect_uri=${DISCORD_OAUTH_REDIRECT_URI}&response_type=code&state=${state}`
3. Return `{ url }`.
4. Permission integer 2147483648 = Send Messages + Read Messages + View Channels.
