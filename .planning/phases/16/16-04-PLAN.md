In the existing startup sync block in `src/index.ts` (the IIFE after `server.listen`), add a separate block AFTER the existing WhatsApp/Telegram `Promise.allSettled(...)` to handle Discord bots.

The existing sync only queries `status: { notIn: ['connected', 'disconnected'] }` which excludes bots with `status: 'connected'`. Discord bots with `status: 'connected'` need to be reconnected because their in-process clients die on server restart (per Pitfall 4 and Assumption A3 in RESEARCH.md).

Add after the existing sync block:

1. Dynamic import `DiscordAdapter` from `./adapters/discordAdapter`
2. Dynamic import `normalizeDiscordMessage` from `./normalizer/discord`
3. Dynamic import `discordMessagesQueue` from `./queue/setup`
4. Query: `prisma.bot.findMany({ where: { platform: 'discord', status: 'connected', sessionName: { not: null } }, select: { id: true, tenantId: true, config: true } })`
5. For each Discord bot:
   - Extract `bot_token` from `(bot.config as any)?.bot_token`
   - If no token, log warn and skip
   - Call `DiscordAdapter.connect(bot_token, bot.id)`
   - Get client via `DiscordAdapter.getClient(bot.id)`
   - Attach `client.on('messageCreate', async (message) => { ... })` that:
     - Returns early if `message.author.bot`
     - Normalizes via `normalizeDiscordMessage(bot.tenantId, message)`
     - Queues via `discordMessagesQueue.add('incoming-message', normalized)`
     - Catches and logs errors
   - Log success
   - On failure: log error, update bot status to 'error' in DB
6. Wrap entire block in try/catch to prevent startup sync failure from crashing server.
